
# 香农编码

## 一、香农编码是什么？

香农编码（Shannon Coding）是信息论创始人克劳德·香农提出的第一种实用的变长编码方法。它根据符号出现的概率大小，为每个符号分配一个二进制码字，使得**大概率符号用短码，小概率符号用长码**，从而平均码长接近信源熵，实现数据压缩。

---

## 二、核心思想

想象我们把区间 $[0,1)$ 按照每个符号的概率大小划分成若干段，每个符号对应一个子区间。然后我们用**子区间起点的二进制小数**的前若干位作为该符号的码字。这样，由于不同符号的区间不重叠，它们的码字也一定不会冲突，而且大概率符号的区间长，需要的二进制位数少；小概率符号的区间短，需要的位数多。

---

## 三、编码步骤（标准流程）

假设信源有 $n$ 个符号 $x_1, x_2, ..., x_n$，对应的概率为 $p_1, p_2, ..., p_n$，且所有概率之和为 $1$。

### 第1步：概率排序
将符号按概率从大到小排列（若概率相等，可任意排序，但顺序一旦固定就不能改变）。  
记排列后的符号为 $s_1, s_2, ..., s_n$，概率仍为 $p_1 \ge p_2 \ge ... \ge p_n$。

### 第2步：计算累积概率
对每个符号 $s_i$，计算累积概率 $P_i = \sum_{k=1}^{i-1} p_k$（即排在它前面所有符号的概率之和）。  
特别地，$P_1 = 0$。

### 第3步：确定码字长度
对每个符号 $s_i$，计算码字长度 $l_i = \lceil -\log_2 p_i \rceil$。  
这里 $\lceil x \rceil$ 表示向上取整，即不小于 $x$ 的最小整数。

> **为什么取这个长度？**  
> 因为长度为 $l$ 的二进制前缀可以区分长度为 $2^{-l}$ 的区间。要能唯一标识一个概率为 $p_i$ 的区间，需要 $2^{-l_i} \le p_i$，即 $l_i \ge -\log_2 p_i$。取最小整数 $l_i$ 既满足要求，又使码长尽可能短。

### 第4步：将累积概率转换为二进制小数，取前 $l_i$ 位作为码字
将累积概率 $P_i$ 表示成二进制小数（例如 $0.1011\ldots$），然后取小数点后的前 $l_i$ 位作为该符号的码字。如果二进制小数不足 $l_i$ 位，则在末尾补 $0$（实际上因为 $P_i$ 是有限小数或循环小数，截取时直接取前 $l_i$ 位即可）。

> **注意**：累积概率 $P_i$ 通常是一个十进制小数，我们需要把它转换成二进制。转换方法是“乘2取整法”：不断乘以2，取整数部分，余数继续乘2，直到得到足够的位数。

## 四、详细例子

### 例：信源符号及概率
设有四个符号 A, B, C, D，概率分别为：
- A: $0.4$
- B: $0.3$
- C: $0.2$
- D: $0.1$

### 步骤1：排序（已按降序排列）
| 符号 | 概率 $p$ |
|------|----------|
| A    | $0.4$    |
| B    | $0.3$    |
| C    | $0.2$    |
| D    | $0.1$    |

### 步骤2：计算累积概率
- A：前面无符号，$P_A = 0$
- B：前面有A（$0.4$），$P_B = 0.4$
- C：前面有A和B（$0.4+0.3=0.7$），$P_C = 0.7$
- D：前面有A、B、C（$0.4+0.3+0.2=0.9$），$P_D = 0.9$

| 符号 | 概率 $p$ | 累积概率 $P$ |
|------|----------|--------------|
| A    | $0.4$    | $0$          |
| B    | $0.3$    | $0.4$        |
| C    | $0.2$    | $0.7$        |
| D    | $0.1$    | $0.9$        |

### 步骤3：计算码字长度
- A: $-\log_2 0.4 \approx 1.3219$，向上取整 → $l_A = 2$
- B: $-\log_2 0.3 \approx 1.7369$，向上取整 → $l_B = 2$
- C: $-\log_2 0.2 \approx 2.3219$，向上取整 → $l_C = 3$
- D: $-\log_2 0.1 \approx 3.3219$，向上取整 → $l_D = 4$

### 步骤4：将累积概率转二进制并取前 $l_i$ 位

#### 符号 A
- 累积概率 $P_A = 0$
- $0$ 的二进制小数是 $0.0000\ldots$  
- 取前 $l_A=2$ 位 → 前两位是 `00`  
- 码字 = **00**

#### 符号 B
- 累积概率 $P_B = 0.4$
- 将 $0.4$ 转为二进制（用乘2取整法）：
  - $0.4 \times 2 = 0.8$ → 整数部分 $0$，余 $0.8$
  - $0.8 \times 2 = 1.6$ → 整数部分 $1$，余 $0.6$
  - $0.6 \times 2 = 1.2$ → 整数部分 $1$，余 $0.2$
  - $0.2 \times 2 = 0.4$ → 整数部分 $0$，余 $0.4$
  - 循环…  
  所以 $0.4$ 的二进制小数是 $0.01100110\ldots$（前几位：$0,1,1,0,0,1,\ldots$）
- 取前 $l_B=2$ 位 → 前两位是 `01`  
- 码字 = **01**

#### 符号 C
- 累积概率 $P_C = 0.7$
- 将 $0.7$ 转二进制：
  - $0.7 \times 2 = 1.4$ → 整数 $1$，余 $0.4$
  - $0.4 \times 2 = 0.8$ → 整数 $0$，余 $0.8$
  - $0.8 \times 2 = 1.6$ → 整数 $1$，余 $0.6$
  - $0.6 \times 2 = 1.2$ → 整数 $1$，余 $0.2$
  - $0.2 \times 2 = 0.4$ → 整数 $0$，余 $0.4$
  - 循环…  
  所以 $0.7$ 的二进制小数是 $0.10110011\ldots$（前几位：$1,0,1,1,0,0,\ldots$）
- 取前 $l_C=3$ 位 → 前三位是 `101`  
- 码字 = **101**

#### 符号 D
- 累积概率 $P_D = 0.9$
- 将 $0.9$ 转二进制：
  - $0.9 \times 2 = 1.8$ → 整数 $1$，余 $0.8$
  - $0.8 \times 2 = 1.6$ → 整数 $1$，余 $0.6$
  - $0.6 \times 2 = 1.2$ → 整数 $1$，余 $0.2$
  - $0.2 \times 2 = 0.4$ → 整数 $0$，余 $0.4$
  - $0.4 \times 2 = 0.8$ → 整数 $0$，余 $0.8$
  - 循环…  
  所以 $0.9$ 的二进制小数是 $0.11100110\ldots$（前几位：$1,1,1,0,0,1,\ldots$）
- 取前 $l_D=4$ 位 → 前四位是 `1110`  
- 码字 = **1110**

### 最终编码表
| 符号 | 概率 | 累积概率 | 码长 | 码字 |
|------|------|----------|------|------|
| A    | 0.4  | 0        | 2    | 00   |
| B    | 0.3  | 0.4      | 2    | 01   |
| C    | 0.2  | 0.7      | 3    | 101  |
| D    | 0.1  | 0.9      | 4    | 1110 |

### 平均码长
$$
L = 0.4\times 2 + 0.3\times 2 + 0.2\times 3 + 0.1\times 4 = 0.8 + 0.6 + 0.6 + 0.4 = 2.4 \text{ 比特/符号}
$$

信源熵：
$$
H = -0.4\log_2 0.4 - 0.3\log_2 0.3 - 0.2\log_2 0.2 - 0.1\log_2 0.1 \approx 1.846 \text{ 比特/符号}
$$
平均码长 $L = 2.4$，大于熵，但小于 $H+1 = 2.846$，符合香农编码的理论界。

## 五、如何解码？

香农编码是**前缀码**（没有任何一个码字是另一个码字的前缀），因此解码时从左到右扫描二进制串，一旦匹配到某个码字，就输出对应符号，然后继续。  
例如上面的码字：`00`, `01`, `101`, `1110`。如果收到 `00101110`，解码过程：
- 读 `00` → A
- 剩下 `101110`，读 `101` → C
- 剩下 `110`，读 `1` ？不对，`1` 不是码字，继续读 `11` 也不是，读 `111` 也不是，读 `1110` → D
- 得到 A, C, D。

## 六、香农编码的优缺点

**优点**：
- 编码方法系统、简单，有严格的数学基础。
- 平均码长接近信源熵，压缩效率高。
- 码字是前缀码，无需分隔符即可唯一解码。

**缺点**：
- 对于某些概率分布，平均码长不如哈夫曼编码（哈夫曼编码总是最优的）。
- 需要对所有符号概率排序，且码长可能偏长（例如上例中D用了4位，而哈夫曼可能用3位）。
- 不适用于动态变化的信源（需要预先知道所有概率）。
